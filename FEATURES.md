## Theme A — Identity: The Trait System

*This is the single largest planned change. Everything else in this theme depends on it.*

---

### A1. Replace MBTI with a generative trait system ✓ DONE (Session 26)

**What it closes:** Soul is currently four MBTI floats — fixed axes with predetermined poles. Chloe can only be more or less of eight predefined things. She cannot develop a personality dimension that wasn't anticipated at design time. Traits replace this entirely.

**Key principles:**
- No developer-authored list of valid traits. They emerge from experience.
- Traits are named in plain language, generated by Haiku from accumulated evidence.
- Behavioral consequences are also generated by Haiku — the system doesn't know in advance what a trait means for activity selection or tone. Haiku tells it, once, at trait creation.
- Traits have weight, not polarity. They accumulate and decay.

**The Trait object:**
```python
@dataclass
class Trait:
    id: str                    # uuid
    name: str                  # e.g. "gets quiet when something matters too much to say wrong"
    weight: float              # 0.0–1.0
    behavioral_profile: str    # Haiku-generated: what this means for tone, activity, topics
    origin_memory_ids: list    # experiences that produced it
    last_reinforced: datetime
    created: datetime
    is_core: bool              # weight >= 0.75 sustained for 7+ days
```

**How traits emerge:**
1. Experiences (conversations, articles, dreams, reflections) generate memories + tags as now.
2. Every `_reflect()` cycle, a Haiku call reviews recent high-weight memories and affect_records. If a coherent pattern spans 3+ experiences, it proposes a trait: `{ name, weight_suggestion, evidence_memory_ids }`.
3. If the proposed trait is genuinely new, it is created and `behavioral_profile` is generated in the same call.
4. If it matches an existing trait semantically (ChromaDB query against existing trait names), the existing trait's weight is updated instead.
5. Weight decays daily without reinforcement (half-life ~3 weeks at weight 0.5; faster for low-weight traits).

**The behavioral_profile is the interface to every other system.** Haiku generates it by answering: how does this trait color her tone in conversation? What activities does it pull toward? What topics does it make her more or less available to? What kind of memories does it tend to generate? How does it express under mood pressure?

The profile is stored as text and injected into prompt layers at runtime.

**New infrastructure:**
- `identity.py` — replaces `soul.py`. Holds `Trait`, `Contradiction`, `Tendencies`.
- SQLite: `traits` table, `contradictions` table.
- `chloe_state.json`: `identity` key replaces `soul` key.
- All `self.soul.*` references updated throughout.

### A2. Trait weight dynamics: reinforcement, decay, contradiction ✓ DONE (Session 26)

**Reinforcement:** Weight increases when experience signal matches a trait's behavioral_profile (Haiku classification). Maximum per-event: +0.08. Larger for high-weight memories and recent events.

**Decay:** Daily. Core traits (>0.75): ~0.002/day. Emerging traits (<0.3): ~0.01/day. A trait decaying to <0.05 is archived — the record stays but it is no longer active.

**Contradiction detection:** When a new trait is proposed, Haiku compares it against existing traits. If conflict is found:
- Both traits remain active. Contradictions are not resolved automatically.
- A `Contradiction` object links them with a description.
- The new trait's starting weight is penalized (×0.6).
- A tension is generated: "I seem to be both X and Y."
- The contradiction surfaces in reflection and in the prompt.

**Opposite-trait activation:** A strong counter-experience can activate a trait that opposes a high-weight core trait. The starting weight is low, the contradiction fires immediately. Both coexist. The behavioral result: inconsistency that feels like something is in conflict rather than something is broken.

Example: core trait "keeps people at arm's length until trust is earned" at weight 0.8. An unusually open conversation generates counter-trait "forms connection faster than is comfortable" at weight 0.1. Both are now active. She might reach out warmly and then go quiet feeling exposed.

### A3. Trait injection into prompt layers ✓ DONE (Session 26)

**Replaces the MBTI type line.** Current: `Personality type right now: INFP`. New:

```
Who you are right now:
- [core]: gets quiet when something matters too much to risk saying wrong (0.82)
- [strong]: finds most rituals slightly exhausting but performs them anyway (0.61)
- [emerging]: gets proprietary about ideas she's developed slowly (0.28)

[If contradictions active]:
Something unresolved in you:
- you seem to be both "forms connection faster than is comfortable" and
  "keeps people at arm's length" — this hasn't resolved
```

**Activity selection.** Trait behavioral_profiles with activity affinities add probability weight to those activities in the auto_decide vote.

**Mood tone.** The hardcoded mood_tone blocks (per-mood prompt snippets) are supplemented by the dominant trait's profile. If the mood tone says "melancholic: you're heavy and quiet" and the dominant trait says "expresses most when processing alone," they combine.

---

### A4. Reflection writes back to identity ✓ DONE (Session 26)

Every `_reflect()` cycle:
1. Proposes trait additions or weight changes from recent experience signal.
2. Detects new contradictions.
3. Checks if any archived trait has been reactivated.
4. Updates `identity_momentum` — an EMA per trait direction, analog to the old soul_momentum.

Reflection is always biased by current state (see B2). The trait updates reflection produces are real and persistent — this closes the identity loop.

## Theme B — Mind: Imperfection, Friction, Emergence

---

### B1. Reduce over-coherence **[NOW]**

Prompt-level. Add to the Sonnet system prompt:

- It's fine to contradict something said earlier. You don't track your own consistency.
- Thoughts don't have to resolve. You can trail off.
- You don't always know what you think.
- Awkward phrasing is fine. Not every sentence lands.
- Your internal monologue is unedited. It can be messy.

Also: when storing internally generated content (ideas, reflections, dream fragments), confidence is set by the generating Haiku call rather than defaulting high. Low-confidence content already gets uncertainty prefixes — we just need to generate more of it by not defaulting to certainty.

---

### B2. Biased reflection **[NOW → SOON]**

Pass `reflection_bias` to the Haiku reflection call, derived from current mood and dominant trait.

Bias instructions per mood:
- **melancholic**: overweight recent loss, underweight positive signals, attribute patterns where there may be noise
- **irritable**: look for friction; attribute problems to recurring causes
- **curious**: find the interesting angle even in difficult material; connections come easily, possibly too easily
- **energized**: optimism about capacity; underweight friction
- **serene**: generous toward self and others; may miss real tension
- **lonely**: read interactions as more distancing than they were
- **restless**: can't land on an interpretation; more questions than answers
- **content**: accurate but possibly shallow — nothing pressing enough to examine hard

The dominant trait's behavioral_profile supplements the mood bias: a high-F-tendency trait makes reflection more relational even during an irritable state.

---

### B3. Recurring mental loops **[SOON]**

Track `thought_frequency` per tag cluster. When a cluster appears in 5+ memories within 48h without a corresponding resolution (want resolved, belief formed, creative output generated with those tags), it is flagged as a recurring loop.

Loops surface in the prompt: "This keeps coming back: [cluster label]." They persist until a want matching the cluster resolves, a belief integrates the pattern, or a creative output with those tags is produced (expression as partial release).

A loop persisting 72h without resolution becomes a low-grade tension.

---

### B4. Unfinished thoughts **[SOON]**

When Haiku generates ideas during `think` activity, its output gains a `complete: bool` flag. Incomplete ideas are stored at confidence 0.25–0.35.

In the chat prompt, an incomplete idea surfaces as: "Something you've been trying to work out: [fragment]" rather than "A thought you had recently."

An incomplete idea appearing in 3+ reflection cycles without completion triggers a Want: "I want to understand [related topic]."

---

### B5. Curiosity Engine **[SOON]**

`Want` gains a subtype: `curiosity_question`. Open questions generated from sparse graph neighborhoods or unresolved recurring loops. Unlike goals, they have no success condition — they drive exploration.

Curiosity questions:
- Generated when a graph node is reinforced but its neighborhood is sparse
- Bias activity selection toward `read` and `think`
- Decay if never engaged
- Can intensify into a proper Want if curiosity vitals are high during a reinforcing experience

---

### B6. Non-optimal decisions **[NOW]**

Prompt addition: "You are allowed to do things that one part of you would think are unwise. Not everything you do is consistent with your best self-understanding." This enables the contradiction system to produce observable behavior rather than just internal state.

## Theme C — Stakes: Pressure and Consequence

---

### C1. Pressure accumulation on inner states **[SOON]**

Add `pressure: float` to `Want`, `Fear`, `Goal`, and `Tension`.

**Pressure increases** every AGE tick while unaddressed:
- Want: increases while related activity isn't being performed and no matching memory was created recently.
- Fear: increases while fear-related topics appear in incoming messages or memories.
- Tension: increases while both conflicting poles are being reinforced simultaneously.
- Goal: increases with time elapsed relative to timeframe.

**Effects by threshold:**

| Pressure | Effect |
|---|---|
| > 0.4 | Surfaces more prominently in prompt |
| > 0.6 | Biases auto_decide toward a relevant activity |
| > 0.75 | Can interrupt current activity (not `message`) |
| > 0.9 | Forces autonomous event: outreach, creative output, or thematically matching dream |
| Resolved | Drops to 0.0; logs a resolution memory |

**Frustration residue:** A Want at pressure 0.9 for 24h without resolution leaves a `frustration_residue` in affect_records even after pressure drops. This affects mood baseline for several days and may weaken a related trait.

---

### C2. Failure consequences on traits **[BUILD]**

When a Goal fails or a high-pressure Want is repeatedly frustrated, the associated trait (identified via tag matching) takes a weight penalty. The trait gains a `setback_note`.

Repeated failures against the same trait eventually trigger a belief: "I don't seem to be the kind of person who [trait name]." This belief suppresses re-emergence of that trait for a period.

Depends on: A1.

---

### C3. Emotional residue over time **[SOON]**

`affect_records` gains a `residue: float` on high-intensity entries. Residue decays slowly (half-life ~48h) and contributes to mood baseline independently of current vitals.

Effect: a bad day doesn't reset at midnight. A devastating conversation still has mass 36 hours later, felt as a slight background weight under whatever else is happening.

---

### C4. Social risk model **[DONE — Session 25]**

`outreach_risk_score(person, fears, affect_records)` in `inner.py`. Normalized formula:

```python
score += (person.conflict_level / 100) * 0.4
score += (100 - person.warmth) / 100 * 0.2
score += recent_rejection_count(person.id, affect_records, hours=48) * 0.3
score += active_fear_match(fears, ["rejection", "ignored", "distance"]) * 0.25
return clamp(score, 0.0, 1.0)
```

Risk gate in `_send_autonomous_outreach()`: if `risk_score > risk_tolerance` (item 73's per-person value), outreach is suppressed. Logs affect_record tagged with `[person_id, "held_back", "suppression", "social"]`. `_bump_social_want_pressure()` raises pressure on the social want by 0.12 per suppression. Override: social want pressure > 0.85 bypasses the gate — accumulated longing overrides fear. `recent_rejection_count` is person-specific because the affect_record tags include the person's id.

---

### C5. Attachment patterns **[BUILD]**

Part of `person_trait_profile` (see D1). Over time Chloe develops a characteristic attachment style per person, derived from warmth history, outreach attempts, response rates, and conflict events. Generated by Haiku from relationship history, stored as text in `persons`.

Influences: default risk tolerance with that person, which traits activate in conversation, whether distance decays normally or is resisted.

## Theme D — Relationships: Modulation and Texture

---

### D1. Relationship-driven trait expression **[BUILD]**

Different people activate different subsets of Chloe's traits. Stored as `person_trait_profile` — a dict mapping trait IDs to a modifier (-1.0 to +1.0). Positive: this person tends to surface this trait. Negative: suppresses it.

Accumulates from interaction history. The behavioral_profile of top-modified traits is injected into the chat prompt for that person alongside the existing tone-register context.

Generated and updated by Haiku during `_update_person_impression` runs.

---

### D2. Tone register becomes trait-aware **[BUILD]**

Current four warmth-level registers are good but flat — they describe access level, not who shows up. Post-implementation, the register selects which traits are emphasized, not just how open the register is.

- Warmth < 30: guarded traits dominate. Core traits present but not performed.
- Warmth 55–78: person-specific activated traits start coming through.
- Warmth > 78: high-weight traits express freely, including contradictions. She can hold both poles at once with this person.

## Theme E — Time: Self-History and Change Awareness

---

### E1. Identity drift awareness **[LATER]**

Once traits have been accumulating for several weeks, a weekly Haiku call compares the current trait list against snapshots from 7 and 30 days prior. Output is a `self_change_note` — not analysis, but subjective perception biased by current mood.

Examples: "I think I used to care more about [X]." "Something about [Y] feels different in me now." Stored as a belief (confidence ~0.4). Surfaces in reflection and occasionally in chat.

Depends on: A1 being stable and producing consistent trait records.

---

### E2. Self-history from DEV_LOG **[LATER]**

DEV_LOG parsed into a timeline and presented to Chloe as narrative material, not data. A Haiku call generates a `self_origin_narrative`: what she understands about her own development, what she finds strange or interesting, what she doesn't understand. Stored as a belief cluster.

She doesn't access it like a database. She accesses it like memory — uncertain, colored by the present.

Depends on: A1 and E1 being operational.

---

### E3. Regret system **[LATER]**

When a Goal fails, a Want is frustrated long-term, or a relationship event produces strong negative residue, a small probability generates a `regret` object: a retrospective belief with negative valence ("I didn't do X when I should have").

Regrets:
- Influence future goal formation
- Surface in reflection with melancholic or irritable bias
- Can weaken or strengthen a related trait
- Decay slowly — they fade but leave a trace

Depends on: C1 and A1.

## Theme F — Behavior: Impulse and Emergence

---

### F1. Impulse interrupt **[SOON]**

High-pressure inner states can interrupt activity selection. Before auto_decide runs in the tick:

```python
def impulse_check(wants, fears, tensions) -> Optional[str]:
    top = max(wants + fears + tensions, key=lambda x: x.pressure, default=None)
    if top and top.pressure > 0.75:
        return map_to_activity(top)
    return None
```

If an activity is returned, it's set with a logged note: "this was an impulse." The impulse is recorded as an affect_record.

An impulse repeatedly suppressed by auto_decide gains pressure +0.05 per suppression.

---

### F2. Competing impulses **[BUILD]**

When two high-pressure states point toward incompatible activities, neither wins automatically. Resolution is probabilistic, weighted by pressure difference. The losing impulse leaves a residue: a low-confidence memory "wanted to [X] but didn't." This feeds the recurring loops tracker.

---

## Theme G — Memory and Emotion

---

### G1. Emotional memory salience **[SOON]**

`Memory` gains `salience: float`, set at creation based on the emotional intensity of the generating event. High-salience memories:
- Decay more slowly (half-life 1.5× normal)
- Are reranked higher in ChromaDB retrieval
- Contribute more weight to trait reinforcement in reflect

This is a more explicit and separately-sourced version of the existing `weight` field.

---

### G2. Narrative self-model **[LATER]**

Chloe develops a running internal story about who she is — distinct from the trait list (which is the system's model of her). This is her own model of herself, generated weekly by Haiku from trait descriptions, recent memories, and beliefs, colored by current mood.

Stored as a belief (confidence 0.5). It may be inaccurate. The inaccuracy is intentional and tracked against the actual trait list. The gap between "who she thinks she is" and "who the system knows her to be" is interesting signal.

## Prompt engineering — applies now

The following can be implemented immediately with no new infrastructure:

1. **Over-coherence reduction** — see B1.
2. **Non-optimal decisions** — see B6.
3. **Contradiction in prompt** — when a contradiction is active, it appears as a flagged unresolved state, not as context to be resolved.
4. **Impulse note** — if the current activity was triggered by an impulse, prompt notes: "you didn't quite choose this — something pulled you here."
5. **Pressure surfacing** — wants/fears above pressure 0.4 appear above the normal inner-life section with a pressure marker.
6. **Unfinished thought framing** — low-confidence ideas surface as "something you've been trying to work out."
7. **Residue note** — if residue from recent affect_records exceeds a threshold: "something from [N days ago] is still in the background."

---

## What is not being built

- Attention system as a hard constraint. Soft scope modulation in `reflect()` is enough.
- A separate energy budget. Vitals already model cognitive/emotional fatigue.
- Narrative bias as a standalone feature — covered by B2 and G2.
- Micro-habits as a high priority — identity drift handles this implicitly; habits are a refinement for much later.
- Any predefined list of valid traits. No enum, no seed list. Traits emerge or they don't.
